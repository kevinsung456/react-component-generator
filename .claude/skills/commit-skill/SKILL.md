## 개요

이 스킬은 스테이징된 변경사항을 분석하여 Conventional Commits 컨벤션을 따르는 한국어 커밋 메시지로 논리적 단위의 커밋들을 생성합니다.

## 절차

### 1단계: 변경사항 분석
먼저 `git diff --cached`로 스테이징된 변경사항을 확인합니다. 변경된 파일 목록, 추가/삭제/수정된 라인 수를 파악합니다.

### 2단계: 논리적 단위로 분류
변경사항을 다음과 같은 논리적 단위로 분류합니다:
- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **refactor**: 코드 리팩토링 (기능 변화 없음)
- **chore**: 빌드, 의존성, 설정 등의 변경
- **docs**: 문서 작성/수정
- **style**: 포맷팅, 세미콜론 등 (코드 로직 변화 없음)
- **test**: 테스트 추가/수정

여러 종류의 변경이 섞여있으면 각각 별도의 커밋으로 나눕니다.

### 3단계: 커밋 메시지 작성

각 커밋마다 다음 형식으로 메시지를 작성합니다:

```
type(scope): 한국어 요약 (50자 이내)

필요한 경우 본문에 상세 설명을 작성합니다.
72자 단위로 줄을 나누어 작성합니다.

- 변경사항 1
- 변경사항 2

변경 사유나 추가 정보가 있으면 포함합니다.
```

**예시:**
```
feat(components): Button 컴포넌트 추가

새로운 Button 컴포넌트를 추가합니다.

- 기본 스타일 지원
- 크기 변형 (small, medium, large) 지원
- disabled 상태 처리
```

```
fix(api): 인증 토큰 만료 처리

만료된 토큰으로 요청할 때 401 에러를 올바르게 처리합니다.
```

### 4단계: 사용자 승인

각 커밋에 대해 다음 정보를 제시하고 승인을 받습니다:
- 커밋 타입과 제목
- 포함될 파일 목록
- 커밋 메시지 본문

사용자는 다음 중 선택할 수 있습니다:
- 승인: 해당 커밋을 진행
- 수정: 커밋 메시지 수정
- 취소: 이 커밋 건너뛰기
- 재분류: 다른 파일을 함께/분리해서 커밋

### 5단계: 커밋 실행

사용자 승인을 받은 커밋들을 순서대로 실행합니다. 각 커밋 후 결과를 보여줍니다.

## 주의사항

- 스테이징된 파일만 커밋합니다 (`git add` 자동 실행 안 함)
- 한국어 커밋 메시지를 작성합니다
- Conventional Commits 형식을 준수합니다
- 논리적으로 관련된 변경사항만 함께 커밋합니다
- 커밋 메시지는 명령형 시제를 사용합니다 ("수정했다" X, "수정한다" O)

## 상황별 사용 예

**상황 1: 새 기능 추가**
```
User: 커밋해줘
→ feat(sidebar): Sidebar 컴포넌트 추가
   fix(types): Sidebar 관련 타입 정의
```

**상황 2: 여러 버그 수정**
```
User: 변경사항 저장해줘
→ fix(auth): 로그인 상태 유지 버그 수정
   fix(form): 폼 제출 중복 방지
   chore(deps): 의존성 버전 업데이트
```

**상황 3: 리팩토링**
```
User: 커밋해줘
→ refactor(utils): 유틸리티 함수 정리
   docs(README): 설치 가이드 추가
```
